# 구글 시트로 참가 신청 받기 — 설정 가이드

초대장 신청 폼에서 제출된 데이터를 **구글 시트에 한 줄씩** 저장하는 방법입니다.
Google Apps Script(구글 앱스 스크립트)로 간단한 웹앱 주소(엔드포인트)를 만들고,
그 주소를 `src/config.ts` 에 붙여넣으면 끝납니다.

소요 시간: 약 5분 · 별도 서버/비용 없음

---

## 저장되는 시트 컬럼

| 신청시각 | 이름 | 핸드폰번호 | 기수 | 동의여부 |
| --- | --- | --- | --- | --- |
| 2026-08-01 14:22:10 | 홍길동 | 010-1234-5678 | 23기 | Y |

- 헤더 행과 `신청` 탭은 첫 제출 시 **자동으로 생성**됩니다.
- 같은 휴대폰 번호로 다시 신청하면 **중복으로 걸러집니다.**

---

## 1단계 — 구글 시트 만들기

1. [sheets.new](https://sheets.new) 에 접속해 새 스프레드시트를 만듭니다.
2. 이름을 알아보기 쉽게 바꿉니다. (예: `Woori 40th 참가신청`)

> 시트 안에 탭을 미리 만들 필요는 없습니다. 스크립트가 `신청` 탭과 헤더를 알아서 만듭니다.

## 2단계 — Apps Script 열기

1. 상단 메뉴에서 **확장 프로그램(Extensions) → Apps Script** 를 클릭합니다.
2. 새 스크립트 편집기가 열립니다. 기본으로 있던 `function myFunction() {}` 코드는 지웁니다.

## 3단계 — 코드 붙여넣기

1. 이 저장소의 [`google-apps-script/Code.gs`](./google-apps-script/Code.gs) 파일 내용을 **전체 복사**합니다.
2. Apps Script 편집기에 **그대로 붙여넣기** 합니다.
3. 상단의 💾(저장) 아이콘 또는 `Ctrl/Cmd + S` 로 저장합니다.
   - 프로젝트 이름을 물어보면 아무 이름(예: `woori40th`)이나 지어줍니다.

## 4단계 — 웹앱으로 배포하기

1. 오른쪽 위 **배포(Deploy) → 새 배포(New deployment)** 를 클릭합니다.
2. 톱니바퀴(⚙️) 아이콘 → **웹 앱(Web app)** 을 선택합니다.
3. 아래처럼 설정합니다.
   - **설명(Description):** 아무거나 (예: `v1`)
   - **다음 사용자로 실행(Execute as):** **나(Me)**
   - **액세스 권한이 있는 사용자(Who has access):** **모든 사용자(Anyone)** ✅ (중요!)
4. **배포(Deploy)** 를 누릅니다.
5. 처음이라면 **권한 승인** 창이 뜹니다.
   - "이 앱은 Google에서 확인하지 않았습니다" 경고가 나오면
     → **고급(Advanced)** → **(안전하지 않은 페이지로) 이동** → **허용(Allow)** 을 눌러 진행합니다.
     (본인이 만든 스크립트라 안전합니다.)
6. 배포가 끝나면 **웹 앱 URL** 이 나옵니다.
   `https://script.google.com/macros/s/AKfyc..../exec` 형태입니다.
   이 주소를 **복사**해 둡니다.

## 5단계 — 초대장에 주소 붙여넣기

1. `src/config.ts` 파일을 엽니다.
2. `GOOGLE_SHEET_ENDPOINT` 값을 방금 복사한 URL 로 바꿉니다.

```ts
export const GOOGLE_SHEET_ENDPOINT =
  'https://script.google.com/macros/s/AKfyc..../exec'
```

3. 저장하고 사이트를 새로고침(개발 서버는 자동 반영)하면, 이제 신청이 **실제 시트에 저장**됩니다.

> 참고: 주소를 넣기 전(=기본 `PASTE_...` 상태)에는 폼이 **테스트 모드**로 동작합니다.
> 실제 저장 없이 성공 화면까지 흐름만 확인할 수 있고, 완료 화면에 "테스트 모드" 안내가 뜹니다.

---

## 잘 됐는지 확인하기

- **엔드포인트 살아있는지:** 복사한 웹앱 URL 을 브라우저 주소창에 그대로 붙여넣어 열어보세요.
  `{"result":"ok","message":"Woori 40th endpoint alive"}` 가 보이면 정상입니다.
- **실제 제출 테스트:** 사이트에서 신청을 한 번 넣어보고, 시트 `신청` 탭에 줄이 쌓이는지 확인합니다.

## 코드를 수정했다면 (다시 배포)

Apps Script 코드를 고친 뒤에는 **새 버전으로 다시 배포**해야 반영됩니다.

- **배포(Deploy) → 배포 관리(Manage deployments) → (연필 아이콘) → 버전: 새 버전(New version) → 배포**
- 이렇게 하면 **URL 은 그대로 유지**되므로 `config.ts` 는 다시 안 고쳐도 됩니다.

---

## 자주 겪는 문제 (트러블슈팅)

| 증상 | 원인 / 해결 |
| --- | --- |
| 제출은 되는데 시트에 안 쌓임 | 4단계에서 **액세스 권한을 "모든 사용자(Anyone)"** 로 했는지 확인 |
| "권한이 필요합니다" 오류 | 4-5단계의 **권한 승인**을 완료했는지 확인 (본인 계정으로) |
| 코드 고쳤는데 반영 안 됨 | **새 버전으로 다시 배포** 필요 (위 "다시 배포" 참고) |
| 항상 "네트워크 오류" 안내 | URL 이 `/exec` 로 끝나는지 확인 (`/dev` 아님). config 값 오타 확인 |
| 한글이 깨져서 저장됨 | 코드는 UTF-8 처리를 하므로 보통 문제 없음. 시트 인코딩은 자동입니다 |

## 보안/개인정보 메모

- 신청 데이터에는 이름·연락처가 포함되므로, 시트 **공유 범위를 최소화**하세요. (준비위원회만)
- 폼에는 "행사 종료 후 파기" 문구가 안내되어 있습니다. 행사 후 시트를 **정리/삭제**해 주세요.
- 웹앱 URL 은 신청을 받는 통로일 뿐 시트 내용을 노출하지 않지만, 외부에 굳이 공개할 필요는 없습니다.
